# PriceMatrix — Управление прайс-листом поставщика

## Что это

Личный инструмент для работы с прайсом одного поставщика.
Загружаю прайс поставщика → связываю с моим прайсом по артикулу → обновляю цены и остатки на сайте carstvo-sna.ru (InSales).

## Интерфейс

Таблица. Строки — товары, сгруппированные по базовому артикулу (варианты размер/цвет свёрнуты). Столбцы — данные из прайса поставщика и моего прайса рядом.

Таблица отвечает на вопросы сразу, без переходов между экранами:

| Вопрос | Как показано |
|---|---|
| Этот товар есть у поставщика? | Цвет строки или иконка |
| Цена изменилась? | Подсветка ячейки, было → стало |
| Товар новый (нет у меня)? | Метка «Новый» + отдельный фильтр |
| Цена нулевая — товар не будет выгружен? | Метка «Без цены» |
| Синхронизация прошла успешно? | Статус с временем последнего обновления |

## Поставщик

Один поставщик — Platina.

**Связка прайсов — два метода (оба применяются):**
1. По артикулу: первое слово из моего артикула = артикул поставщика
   - Поставщик: `ми4447`
   - Мой прайс: `ми4447 l бело-голубой` → базовый `ми4447`
2. По штрихкоду: точное совпадение col[O] поставщика и col[24] моего прайса

**Статистика (актуальные файлы):**
- Артикулов у поставщика: 2 191
- Базовых артикулов в моём прайсе: 2 567
- Строк в моём прайсе: 28 773
- Строк с ценой > 0: 2 719 | с нулевой ценой: 26 053
- Строк со штрихкодом: 6 671

## Что умеет система

### Загрузка прайса поставщика
- Ручная загрузка файла с компьютера: CSV, Excel, YML, XML
- Предпросмотр данных до сохранения — видно что загрузилось
- Настройка маппинга колонок один раз: система запоминает структуру файла
- Получение прайса через API поставщика (позже)

**Правила парсинга файла Platina (ostatki-Platina.xlsx):**
- Пропускать шапку: строки 0–6, данные начинаются со строки 7
- Фильтровать строки-группы (заголовки категорий): строки без значения в колонке цены
- Колонки: A(0)=Артикул, D(3)=Название, N(13)=Размер+цвет, O(14)=Штрихкод, Q(16)=Цена, R(17)=Остаток
- Цена — всегда из колонки **Q ("6 ПЛАТИНОВЫЙ")**
- Один артикул = несколько строк (варианты по размеру и цвету) → группировать

**Колонки моего прайса (price-carstvo-sna.xls):**
- col[0]=ID товара, col[1]=Название, col[22]=ID варианта
- col[23]=Артикул, col[24]=Штрих-код, col[26]=Цена продажи, col[29]=Остаток
- Цены хранятся как строка `"3596,0"` → нормализовать в число при импорте

### Таблица товаров (PriceMatrix)
- Строки сгруппированы по базовому артикулу, варианты (размер/цвет) — свёрнуты/развёрнуты
- Слева — мой прайс, справа — данные поставщика, связаны по артикулу или штрихкоду
- Видно: цена поставщика ("6 ПЛАТИНОВЫЙ"), наценка, итоговая цена продажи, остаток
- Визуальные статусы строк:
  - Новый товар (есть у поставщика, нет у меня)
  - Изменилась цена (подсветка, было → стало)
  - Нет у поставщика (товар пропал из прайса)
  - Нулевая цена — в выгрузку не идёт
- Уведомление об ошибках синхронизации

### Цены и наценка
- Цена поставщика — всегда "6 ПЛАТИНОВЫЙ"
- Наценка от цены поставщика: фиксированная (руб.) или процентная (%)
- Итоговая цена продажи видна в таблице до выгрузки
- Товары с нулевой итоговой ценой в InSales не выгружаются
- Только рубли

### Выгрузка
- Обновление цен и остатков существующих товаров в InSales через API
- Новые товары поставщика → формируются в отдельный прайс для выгрузки на сайт
- Экспорт таблицы в Excel, CSV

## Вариант реализации: Google Таблицы

**Плюсы:**
- Не нужна разработка бэкенда и базы данных
- Доступ с любого устройства без установки
- Привычный интерфейс таблицы
- Автоматизация через Google Apps Script (загрузка файлов, InSales API)
- Быстрый старт

**Минусы:**
- Лимит ~5 млн ячеек (28 773 строк × 46 колонок ≈ 1,3 млн — вписывается, но с запасом)
- Тормозит при большом объёме данных и сложных формулах
- Нет виртуализации — все строки рендерятся сразу
- Сложнее реализовать группировку вариантов
- Парсинг xlsx/YML/XML требует Apps Script или внешнего сервиса

**Вывод:** Google Таблицы подойдут как MVP для проверки логики, но при росте данных упрутся в производительность. Веб-приложение — долгосрочное решение.

## Чего не будет

- Несколько поставщиков — только один
- Мультивалюта — только рубли
- Автоматический парсинг сайтов
- Управление заказами, складом, логистикой
- Мобильная версия
