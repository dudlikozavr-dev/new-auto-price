# PriceMatrix — Управление прайс-листом поставщика

## Что это

Личный инструмент для работы с прайсом одного поставщика.
Загружаю прайс поставщика → связываю с моим прайсом по артикулу → обновляю цены и остатки на сайте carstvo-sna.ru (InSales).

## Интерфейс

Таблица. Строки — товары, сгруппированные по базовому артикулу (варианты размер/цвет свёрнуты). Столбцы — данные из прайса поставщика и моего прайса рядом.

Таблица отвечает на вопросы сразу, без переходов между экранами:

| Вопрос | Как показано |
|---|---|
| Этот товар есть у поставщика? | Цвет строки или иконка |
| Цена изменилась? | Подсветка ячейки, было → стало |
| Товар новый (нет у меня)? | Метка «Новый» + отдельный фильтр |
| Цена нулевая — товар не будет выгружен? | Метка «Без цены» |
| Синхронизация прошла успешно? | Статус с временем последнего обновления |

## Поставщик

Один поставщик — Platina.

**Связка прайсов — два метода (оба применяются):**
1. По артикулу: первое слово из моего артикула = артикул поставщика
   - Поставщик: `ми4447`
   - Мой прайс: `ми4447 l бело-голубой` → базовый `ми4447`
2. По штрихкоду: точное совпадение col[O] поставщика и col[24] моего прайса

**Статистика (актуальные файлы):**
- Артикулов у поставщика: 2 191
- Базовых артикулов в моём прайсе: 2 567
- Строк в моём прайсе: 28 773
- Строк с ценой > 0: 2 719 | с нулевой ценой: 26 053
- Строк со штрихкодом: 6 671

## Что умеет система

### Загрузка прайса поставщика
- Ручная загрузка файла с компьютера: CSV, Excel, YML, XML
- Предпросмотр данных до сохранения — видно что загрузилось
- Настройка маппинга колонок один раз: система запоминает структуру файла
- Получение прайса через API поставщика (позже)

**Правила парсинга файла Platina (ostatki-Platina.xlsx):**
- Пропускать шапку: строки 0–6, данные начинаются со строки 7
- Фильтровать строки-группы (заголовки категорий): строки без значения в колонке цены
- Колонки: A(0)=Артикул, D(3)=Название, N(13)=Размер+цвет, O(14)=Штрихкод, Q(16)=Цена, R(17)=Остаток
- Цена — всегда из колонки **Q ("6 ПЛАТИНОВЫЙ")**
- Один артикул = несколько строк (варианты по размеру и цвету) → группировать

**Колонки моего прайса (price-carstvo-sna.xls):**
- col[0]=ID товара, col[1]=Название, col[22]=ID варианта
- col[23]=Артикул, col[24]=Штрих-код, col[26]=Цена продажи, col[29]=Остаток
- Цены хранятся как строка `"3596,0"` → нормализовать в число при импорте

### Таблица товаров (PriceMatrix)
- Строки сгруппированы по базовому артикулу, варианты (размер/цвет) — свёрнуты/развёрнуты
- Слева — мой прайс, справа — данные поставщика, связаны по артикулу или штрихкоду
- Видно: цена поставщика ("6 ПЛАТИНОВЫЙ"), наценка, итоговая цена продажи, остаток
- Визуальные статусы строк:
  - Новый товар (есть у поставщика, нет у меня)
  - Изменилась цена (подсветка, было → стало)
  - Нет у поставщика (товар пропал из прайса)
  - Нулевая цена — в выгрузку не идёт
- Уведомление об ошибках синхронизации

### Цены и наценка
- Цена поставщика — всегда "6 ПЛАТИНОВЫЙ"
- Наценка от цены поставщика: фиксированная (руб.) или процентная (%)
- Наценка по категориям: разные правила для разных групп товаров (пижамы, сорочки, туники и т.д.)
- Итоговая цена продажи видна в таблице до выгрузки
- Товары с нулевой итоговой ценой в InSales не выгружаются
- Только рубли

### История цен поставщика
- При каждом импорте сохранять цену поставщика с датой
- В таблице показывать: текущая цена и предыдущая (было → стало)
- История изменений по каждому артикулу: когда и насколько менялась цена

### Выгрузка
- Обновление цен и остатков существующих товаров в InSales через API
- Новые товары поставщика → формируются в отдельный прайс для выгрузки на сайт
- Экспорт таблицы в Excel, CSV

## Два варианта реализации

Оба варианта реализуются параллельно. Логика одна — оболочки разные.

### Вариант А — Google Таблица (быстрый старт)

Цель: рабочий инструмент за несколько дней, проверка логики на реальных данных.

**Что делает:**
- Загрузка прайса поставщика через Apps Script (импорт xlsx)
- Связка по артикулу и штрихкоду формулами
- Наценка формулой от цены поставщика
- Фильтр нулевых цен
- Выгрузка обновлённых цен и остатков в InSales через Apps Script + API

**Плюсы:**
- Быстрый старт — не нужен бэкенд и база данных
- Привычный интерфейс таблицы
- Доступ с любого устройства
- Легко менять формулы наценки

**Ограничения:**
- Тормозит при большом объёме и сложных формулах
- Группировка вариантов — через вспомогательные колонки, не так красиво
- Парсинг нестандартной шапки Platina требует Apps Script

---

### Вариант Б — Веб-приложение (основное)

Цель: полноценный инструмент с нормальным UX, виртуализацией таблицы, удобной группировкой вариантов.

**Стек:** React + AG Grid, Node.js, PostgreSQL, InSales API

**Плюсы:**
- Быстрая таблица с 15 000+ строк (виртуализация)
- Удобная группировка вариантов
- Полный контроль над интерфейсом
- Надёжнее при росте данных

**Ограничения:**
- Требует разработки бэкенда и настройки базы данных

---

### Порядок работы

1. **Сначала** — Google Таблица: проверить логику связки и наценки на живых данных

## Защита от рисков

### Риск 1 — Поставщик изменил структуру файла
**Последствие без защиты:** парсер читает неправильные колонки, на сайт выгружаются кривые данные.

**Защита:**
- После парсинга показывать сводку: сколько строк прочитано, сколько отфильтровано, первые 5 строк с данными — пользователь видит результат до сохранения
- Проверять что колонка цены содержит числа, артикул — строки начинающиеся с "ми" (для Platina)
- Если количество распознанных строк резко меньше предыдущего импорта (например, меньше 1 000 при ожидаемых 2 000+) — показывать предупреждение

### Риск 2 — Выгрузка в InSales упала на середине
**Последствие без защиты:** часть товаров обновлена, часть нет — сайт в несогласованном состоянии.

**Защита:**
- Перед выгрузкой сохранять список всех товаров которые будут изменены
- После выгрузки показывать отчёт: обновлено X, ошибок Y, список с ошибками
- При ошибке — возможность повторить только для несохранённых товаров
- Соблюдать лимит InSales API: 500 запросов за 5 минут (пакетная обработка с паузами)

### Риск 3 — Аномальная цена у поставщика
**Последствие без защиты:** на сайте появляется товар за 1 руб. или 999 999 руб.

**Защита:**
- Задать допустимый диапазон цен: минимум 100 руб., максимум 50 000 руб. (настраивается)
- Товары вне диапазона — помечаются в таблице, в выгрузку не идут без ручного подтверждения
- Если цена изменилась более чем на 50% по сравнению с предыдущим импортом — предупреждение

### Риск 4 — Наценка применилась незаметно к изменившейся цене
**Защита:**
- В таблице всегда показывать три колонки рядом: цена поставщика / наценка / итоговая цена
- Подсвечивать строки где итоговая цена изменилась с прошлого раза

## Чего не будет

- Несколько поставщиков — только один
- Мультивалюта — только рубли
- Автоматический парсинг сайтов
- Управление заказами, складом, логистикой
- Мобильная версия
