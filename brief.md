# Техническое задание: PriceMatrix
## Управление прайс-листом поставщика

---

## 1. Цель и аудитория

### Цель
Веб-приложение для владельца интернет-магазина: загрузить прайс поставщика, поставить наценку, выгрузить обновлённые цены и остатки в InSales — без ручной работы в Excel.

### Целевая аудитория
**Основной пользователь:** владелец небольшого интернет-магазина на InSales.
- Работает с 1–2 поставщиками
- Обновляет прайсы вручную раз в неделю, тратит 2–3 часа
- Не программист — слова "API", "база данных", "деплой" его пугают
- Устал копировать цены из Excel на сайт

**Боль:** каждую неделю вручную переносить цены и остатки из файла поставщика на сайт.
**Желаемый результат:** загрузил файл → нажал кнопку → цены обновились.

---

## 2. Структура приложения

Приложение — одностраничное (SPA). Навигация минимальная: три раздела в сайдбаре или верхнем меню.

```
┌─────────────────────────────────────────────────┐
│  PriceMatrix          [Статус: обновлено 14:32] │
├──────────┬──────────────────────────────────────┤
│          │                                      │
│  Товары  │         Основная область             │
│  Импорт  │         (таблица / wizard)           │
│  Настрой.│                                      │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

### Три раздела

| Раздел | URL | Назначение |
|---|---|---|
| Товары | `/` | Главная таблица: мой прайс + данные поставщика |
| Импорт | `/import` | Загрузка нового файла поставщика |
| Настройки | `/settings` | Наценка по категориям, токен InSales, диапазон цен |

## 3. Раздел «Импорт» — Wizard загрузки файла

Весь флоу на одной странице. Четыре шага без переходов между экранами.

### Шаг 1 — Загрузка файла

**Состояние по умолчанию (пустое):**
```
┌─────────────────────────────────────┐
│                                     │
│   📂  Перетащи файл сюда            │
│       или нажми для выбора          │
│                                     │
│   Поддерживаются: xlsx, xls, csv    │
│                                     │
└─────────────────────────────────────┘
```

- Drag-and-drop зона с пунктирной рамкой
- При наведении файла — рамка подсвечивается синим
- Принимает: `.xlsx`, `.xls`, `.csv`
- После выбора — сразу переходит к шагу 2 (без кнопки "Далее")

**Текст:** "Загрузи прайс-лист поставщика. Файл не изменится — мы только читаем данные."
*(Снижение страха: "не сломается")*

---

### Шаг 2 — Предпросмотр и маппинг колонок

Показывается сразу после загрузки, без ожидания.

**Верхняя часть — сводка файла:**
```
✓ Файл загружен: ostatki-Platina.xlsx
  Строк с данными: 2 191  •  Пропущено строк-групп: 340  •  Пропущено шапки: 7 строк
```

**Маппинг колонок** (select-mapper):

| Колонка в файле | Содержимое (пример) | Поле системы |
|---|---|---|
| A | ми4447 | [Артикул ▼] ← автоопределено |
| D | Пижама Mia-Amore 4447 | [Название ▼] ← автоопределено |
| N | S сливовый | [Размер+цвет ▼] ← автоопределено |
| O | 2170444718910 | [Штрихкод ▼] ← автоопределено |
| Q | 3 598 | [Цена поставщика ▼] ← автоопределено |
| R | 1 | [Остаток ▼] ← автоопределено |

- Автоопределение по заголовкам файла ("6 ПЛАТИНОВЫЙ" → цена поставщика)
- Уже привязанные поля — зелёная галочка
- Обязательные непривязанные — красная рамка
- Маппинг сохраняется, при следующей загрузке применяется автоматически
- Кнопка "Сбросить до автоопределения"

**Кнопка:** "Проверить данные →"

## 9. Технические требования

### Frontend
- **Фреймворк:** React 18+
- **Таблица:** AG Grid Community (MIT) — обязательно, не заменять
- **Стили:** Tailwind CSS или CSS Modules
- **Роутинг:** React Router v6
- **Состояние:** React Query для серверных данных + Zustand для UI-состояния
- **Виртуализация:** встроенная в AG Grid (не добавлять внешнюю)

### Backend
- **Runtime:** Node.js 20+
- **Framework:** Express
- **База данных:** PostgreSQL
- **Парсинг Excel:** библиотека `xlsx`
- **Очередь задач:** встроенная (в памяти для MVP, Bull/BullMQ для продакшена)
- **Real-time прогресс:** SSE (Server-Sent Events)

### API
- `POST /api/import` — загрузка файла (multipart/form-data)
- `GET  /api/import/preview/:id` — предпросмотр изменений
- `POST /api/import/confirm/:id` — подтвердить импорт
- `GET  /api/items` — список товаров (с пагинацией/фильтрами)
- `PUT  /api/items/:id/markup` — изменить наценку
- `POST /api/export/insales` — запустить выгрузку в InSales
- `GET  /api/export/insales/:jobId` — статус выгрузки (SSE)
- `GET  /api/settings` / `PUT /api/settings` — настройки

### InSales API
- Авторизация: Basic Auth (API key + password)
- Лимит: 500 запросов / 5 минут → темп 400 req/min
- Стратегия: карта `sku → {product_id, variant_id}` в БД, обновлять только изменившиеся

## 10. Поток данных

```
[файл поставщика]
       ↓ multipart upload
[POST /api/import]
       ↓ xlsx-парсер
       ↓ пропуск шапки (строки 0–6)
       ↓ фильтрация строк-групп (нет цены в col Q)
       ↓ нормализация: "3598,0" → 3598
[таблица supplier_items в PostgreSQL]
       ↓ сопоставление: базовый артикул + штрихкод
[таблица matches: supplier_sku ↔ my_sku]
       ↓ применение наценки по категории
[итоговые цены продажи]
       ↓ diff с предыдущим состоянием
[предпросмотр изменений → подтверждение]
       ↓ очередь выгрузки (SSE прогресс)
[InSales API: PUT /variants/:id → цена + остаток]
```

## 11. Открытые вопросы (требуют решения до разработки)

1. **Категории товаров** — в Platina группы называются по коллекциям (`Камея`, `Лилианна`). Это коллекции или категории для наценки? Определить до разработки раздела настроек.
2. **Деплой** — где будет работать: локально / VPS / Railway? Влияет на архитектуру очереди и хранилища файлов.
3. **Токен InSales** — хранить в `.env` файле, не в базе данных.
4. **Архив прайсов** — при загрузке нового файла сохранять предыдущий с датой (папка `/uploads/archive/`).
5. **Один склад или несколько** — влияет на эндпоинт обновления остатков в InSales.
