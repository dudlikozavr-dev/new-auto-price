# Правила проекта PriceMatrix

## Поведение

- После каждого изменения файлов — коммит и пуш без напоминания
- Отвечать кратко и по делу
- Не добавлять комментарии в код если не попросили
- Не создавать новые файлы без необходимости — лучше редактировать существующие
- Не добавлять функции которые не были запрошены

## Коммиты

- Сообщение коммита — на английском, одна строка
- Всегда добавлять `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>`

## Стек

- Backend: Node.js (Express) — не Python
- Frontend: React + AG Grid
- База данных: PostgreSQL
- Парсинг Excel: библиотека xlsx

## Бизнес-правила

- Цена поставщика — всегда колонка Q ("6 ПЛАТИНОВЫЙ"), никакая другая
- Товары с нулевой или пустой ценой — в InSales не выгружать
- Базовый артикул — первое слово из строки артикула (например `ми4447` из `ми4447 l бело-голубой`)
- Связка товаров — сначала по артикулу, затем по штрихкоду
- Валюта — только рубли

## Файлы прайсов

- `ostatki-Platina.xlsx` — прайс поставщика, не редактировать вручную
- `price-carstvo-sna.xls` — прайс магазина, не редактировать вручную
- Шапка прайса Platina — строки 0–6, данные с строки 7
- Строки-группы (без цены) — фильтровать при импорте

## Запреты

- Не менять структуру базы данных без обсуждения
- Не пушить сломанный код
- Не удалять файлы прайсов
